### 方案一：DNS 智能解析（推荐，最常用）

国内主流 DNS 服务商都支持**分线路解析**，为同一条 SRV 记录按地域返回不同 IP：

```
# 假设你的 SRV 域名是 _mongodb._tcp.mongo.ericyonng.com

【默认/海外线路】→ 解析到国外 mongos 组
_0 5 27021 mongos-us1.mongo.ericyonng.com.
_0 5 27021 mongos-us2.mongo.ericyonng.com.

【中国电信/联通/移动线路】→ 解析到国内 mongos 组
_0 5 27021 mongos-cn1.mongo.ericyonng.com.
_0 5 27021 mongos-cn2.mongo.ericyonng.com.
```

国内客户端查询同一域名时，DNS 根据来源 IP 自动返回不同的 SRV 结果。

| 服务商     | 产品名                                   |
| ---------- | ---------------------------------------- |
| 阿里云     | 云解析DNS → 智能解析（分线路）           |
| 腾讯云     | DNSPod → 智能解析/分运营商分地域         |
| Cloudflare | **Geo Steering** (付费) / Load Balancing |

---

### 方案二：双域名

不用智能解析，直接部署两套域名，业务层根据环境选择：

```
# 国内服务配置
MONGO_URI=mongodb+srv://mongo-cn.ericyonng.com

# 海外服务配置
MONGO_URI=mongodb+srv://mongo-global.ericyonng.com
```

缺点：需要业务感知地域，不如方案一透明。

---

### 方案三：Anycast DNS（适用于自建 DNS）

自己搭建 BGP Anycast DNS 集群，同一 DNS 地址在不同区域路由到不同解析节点，每个节点返回本地 SRV 结果。实现复杂度最高，一般不推荐。

---

### 操作步骤

**1. 进入控制台**

登录 [DNSPod 控制台](https://console.dnspod.cn) → 我的域名 → 点你的域名 → **解析设置**。

**2. 添加两条 SRV 记录**

添加时记录类型选 **SRV**，关键在"线路"字段：

```
# 记录1 —— 国内用户
主机记录: _mongodb._tcp.mongo     优先级: 0    权重: 5    端口: 27021
目标: mongos-cn1.mongo.ericyonng.com.
线路: 境内                          ← 国内用户解析到这里
TTL: 600

# 记录2 —— 海外用户  
主机记录: _mongodb._tcp.mongo     优先级: 0    权重: 5    端口: 27021
目标: mongos-us1.mongo.ericyonng.com.
线路: 境外                          ← 海外用户解析到这里
TTL: 600
```

> ⚠️ 注意 SRV 目标的 FQDN 末尾必须带 `.`（如 `mongos-cn1.mongo.ericyonng.com.`）

**3. 验证**

```bash
# 国内机器
dig SRV _mongodb._tcp.mongo.ericyonng.com
# → 应该返回 mongos-cn1...

# 海外机器
dig SRV _mongodb._tcp.mongo.ericyonng.com  
# → 应该返回 mongos-us1...
```
