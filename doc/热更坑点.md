# windows

* windows 下加载dll，会锁文件，所以编译新的dll无法覆盖旧的dll(如果旧的dll没被卸载)
* windows 下LoadLibrary dll后会记住完整的路径名, 如果下次还没卸载旧的，再次加载同一路径下的该dll会引用计数+1
* windows 下热更的终极方案: 每次编译dll时, 自带时间戳比如a.xxx.dll, xxx是时间戳, 热更的时候自动加载xxx最大的, 如果fix 后带指定的xxx，比如fix xxx, 可以指定的加载a.xxx.dll

# Linux

linux 下同路径下同名文件的so, 只要和旧的so内容不一样，那么就会被认为是不同文件， 会重新加载，而且linux下不会锁文件，所以linux下热更比较简单，只要覆盖so重新加载该so即可
