# 规整后

* 主线程
  * KillMonitorMgr: 监控关服
  * 内存日志, 流量信息
  * CommandMgr: 监控输入命令(windows下会启一个线程监控, 因为windows下回阻塞监听命令, linux下扫描文件名)
* 重量级线程池(四分之一cpu核数量, 至少一个线程)
  * LibLog:打印日志
  * FileChangeManager:文件变更热更, 轻量级文件热更
* 轻量级线程(二分之一cpu核数量, 至少一个线程)
  * mongodb 业务线程
  * CenterMemoryCollector(中央内存收集器), GarbageThread(垃圾回收)
* 网络io线程(一个IocpTcpPoller/EpollTcpPoller 两个线程)
  * 接受连接, 收发网络消息等
* 业务线程(每个service一个线程)
